## 1. BL에서 Q의 역할

Black-Litterman 핵심 공식:

```
μ_BL = π + τΣP'(PτΣP' + Ω)⁻¹(Q - Pπ)
```

| 기호 | 의미 | 단위 |
|------|------|------|
| π | CAPM 균형 기대수익 | 월별 수익률 |
| P | 뷰 벡터 (저위험 +, 고위험 −, 시총 가중) | — |
| **Q** | **"내 뷰: 이 포트폴리오가 낼 수익률"** | **월별 수익률** |
| **Ω** | **뷰 불확실성** = τ × P'ΣP (기본값) | **분산** |
| τ | BL 스케일 파라미터 (보통 0.05~0.1) | — |

### Q - P·π 의 의미: "CAPM 대비 초과 뷰"

```
Q - P·π > 0  → BL이 저위험 tilt 강화 (의도한 방향)
Q - P·π = 0  → CAPM과 동일 (뷰 없음)
Q - P·π < 0  → BL이 저위험 tilt 약화 또는 역전 ← 위험!
```

**Q는 반드시 P·π와 같은 단위(월별 수익률)여야 한다.**

### P·π의 부호: Q가 작아도 저변동 우대가 일어나는 이유

CAPM(π)은 고변동 종목에 더 높은 기대수익을 부여한다. P는 저위험 그룹에 +, 고위험 그룹에 -를 부여하므로, 내적 P·π는 음수가 된다.

```
diff = Q - P·π = 0.003 - (음수) = 양수 (Q보다 큰 값)
```

즉, **방향(저변동 우대)은 P·π 보정에서 이미 나온다.** Q는 그 강도를 추가로 조절하는 하이퍼파라미터다. Q=0이어도 diff가 양수이므로 완전히 뷰 없음 상태(Q=P·π)와는 다르다.

| Q 값 | diff = Q - P·π | 결과 |
|------|----------------|------|
| 양수 (작게) | 작은 양수 | 저변동 우대, 분산 유지 ✅ |
| 양수 (크게) | 큰 양수 | 저변동 과집중 → 변동성 급등 ❌ |
| 0 | P·π 절댓값만큼 양수 | 여전히 저변동 우대 (뷰 약화) |
| 음수 | 역방향 가능 | BL이 고위험으로 기울어짐 ❌ |

**저변동 우대 전략은 Q 방향(양수)보다 Q 강도(크기)가 더 중요한 설계 변수다.**

## 2. 핵심 문제: Q_hist/Q_momentum이 왜 실패했나

### 2011-2025 저위험 이상현상 역전

강세장·테크 불마켓에서는 고변동성 종목이 오히려 더 올랐다:

```
P·R_t = (저위험 그룹 수익) - (고위험 그룹 수익)
      = 음수가 빈번
```

| 방법 | 윈도우 | 결과 | 문제 |
|------|--------|------|------|
| Q_hist | 60개월 | 거의 항상 양수이지만 0에 가까움 | 과집중 유발, 변동성 ↑ |
| Q_momentum | 12개월 | 최근 추세 반영 → 음수 빈번 | Q < P·π → BL 역방향 작동 → MDD -42% 폭발 |

**핵심**: Q가 음수가 되면 `Q - P·π < 0`이 되어 BL이 고위험 종목으로 기울어진다. 저위험 전략이 고위험 전략으로 뒤집히는 것이다.

## 3. 월별 데이터 + 일별 공분산 설계

### 설계 목적

월별 수익률만 사용하면 N=500일 때 p/T = 500/60 = 8.3 → 공분산 행렬 추정 불안정.  
일별 수익률로 Σ를 추정하면 T가 1260으로 늘어나 p/T = 0.40으로 개선된다.

### 올바른 단위 처리

```python
# 1. 일별 수익률로 Σ 추정 (학습 기간 60개월 × 21일 = 1260일)
daily_rets = price_close.pct_change().dropna()
Sigma_daily = LedoitWolf().fit(daily_rets.tail(1260)).covariance_

# 2. 월별 단위로 스케일링 (IID 근사)
Sigma_monthly = Sigma_daily * 21

# 3. π, Q, μ_BL은 전부 월별 수익률 단위 유지 → 단위 일관성 확보
```

### 근사 오차 인식

| 조건 | Σ_daily × 21 vs 실제 Σ_monthly |
|------|-------------------------------|
| 일반 기간 | 근사적으로 성립 |
| 위기/고변동성 | 실제 Σ_monthly > 21 × Σ_daily (ARCH 효과, 상관관계 급등) |
| 저변동성 기간 | 실제 Σ_monthly ≈ 21 × Σ_daily (비교적 정확) |

ARCH 효과가 85% 종목에서 확인되므로(03_VolatilityEDA) 스케일링 오차가 존재한다.  
그러나 Ledoit-Wolf 수축(α ≈ 0.2~0.3)을 병용하면 1차 근사로 충분히 합리적이다.

시장 상황과 무관하게 일정한 강도를 유지하되, **선택 자체는 매월 과거 성과만으로 결정** → 사후 선택 편향 없음.

**Q_OPTIMAL 탐색 방식**:

| 방식 | 설명 | 편향 여부 |
|------|------|---------|
| 기존 (전 기간 in-sample) | 전 기간 수익률 본 후 Q 소급 선택 | **사후 선택 편향 있음** (소폭) |
| 현재 (expanding window) | 매월 과거 데이터만으로 Q 선택 | **편향 없음** — 진짜 OOS |

초기 24개월 warmup 이후 Q는 수렴하는 경향이 있다. 역사가 쌓일수록 expanding Sharpe가 안정되기 때문이다.

**장점**: 단순하고 견고함. expanding window로 OOS 신뢰성 확보.  
**한계**: 시장 레짐 변화에 무반응. 위기 때도 동일한 뷰 강도 유지 → 방어력 부족.

**성과**: Sharpe 0.931~1.074 (노트북별 차이), MDD -13.70% ~ -18.81%  
(expanding window 전환 후 수치는 재실행 필요)

### 4-2. Q_hist

```python
def compute_Q_hist(P, ret_matrix):
    # ret_matrix: 학습 기간(60개월) 월별 수익률 행렬 (T × N)
    p_vec = P.reindex(ret_matrix.columns).fillna(0).values
    return float((ret_matrix.values @ p_vec).mean())
    # = mean(저위험 그룹 수익 - 고위험 그룹 수익) over 60M
```

**의미**: "학습 기간 60개월 동안 실제로 관찰된 저위험-고위험 수익 스프레드 평균"을 Q로 사용.  
과거가 미래를 대표한다는 가정.

**장점**: 데이터 기반, look-ahead bias 없음.  
**한계**:
- 2011-2025 강세장에서 스프레드가 0 근처 또는 음수 → Q 과소평가 → 과집중 유발
- 60개월 평균이라 최근 시장 변화에 둔감

**성과**: Sharpe 0.896, MDD -21.57% (Q_FIXED 대비 Sharpe 열세, 변동성 ↑)

### 4-3. Q_momentum

```python
def compute_Q_momentum(P, ret_matrix, window=12):
    # 최근 12개월만 사용
    n = min(window, len(ret_matrix))
    p_vec = P.reindex(ret_matrix.columns).fillna(0).values
    return float((ret_matrix.values[-n:] @ p_vec).mean())
```

**의미**: Q_hist와 공식은 동일하되 최근 12개월 스프레드만 반영.  
"최근 추세가 지속된다"는 단기 모멘텀 가정.

**장점**: 시장 변화에 민감하게 반응.  
**한계**: **민감함이 독이 된다.**
- 고변동성 종목이 강세인 달이 이어지면 Q가 음수로 전환
- Q - P·π < 0 → BL이 고위험 방향으로 기울어짐 → 전략 역전
- MDD -42.00%, Sharpe 0.597 (최하위)

**교훈**: 단기 스프레드는 신뢰성이 낮다. 음수 방지 장치 없이 쓰면 위험하다.

### 4-4. Q_lambda ★ 07 노트북 최고 성과

```python
def compute_Q_lambda(lam, q_base, lam_mean=2.5):
    scale = np.clip(lam / lam_mean, 0.1, 3.0)
    return q_base * scale

# lam: 학습 기간 SPY 초과수익 / SPY 분산 (시장 위험회피계수 추정값)
# lam_mean = 2.5 (He & Litterman 2002 장기 평균)
```

**의미**: 위험회피계수 λ로 Q 강도를 조절.

```
시장 안정 (수익 ↑, 변동성 ↓) → λ ↑ → Q 강화 → 저위험 tilt 강하게
시장 불안 (수익 ↓, 변동성 ↑) → λ ↓ → Q 약화 → 포트폴리오 보수적으로
```

**장점**:
- 역사적 스프레드 사용 안 함 → 음수 문제 없음
- 시장 상황에 자동 적응
- clip(0.1, 3.0)으로 극단값 방지

**한계**: λ는 SPY 수익률/분산으로 추정하므로 시장 전체 위험회피를 대리하는 값. 저위험 프리미엄과 λ의 관계가 항상 성립하지 않을 수 있다.

**성과**: Sharpe 1.112, MDD -11.84% (07 노트북 기준 전체 1위)

### 4-8. raw_lam_Q — raw λ 부호 기반 자연 게이팅

```python
lam_raw = spy_excess_ret / sigma2_mkt   # clip 전 원본 λ (음수 가능)
Q = max(0.0, Q_OPTIMAL * (lam_raw / LAM_MEAN))
```

**λ 설계 배경 — clipped λ vs raw λ**:

`compute_pi`에서 반환하는 `lam`은 π 계산 안정성을 위해 `[0.5, 10.0]`으로 clip된 값이다.

```
π = λ × Σ × w_mkt  →  λ < 0이면 π 음수 → MVO 붕괴 → clip 필수
```

그러나 이 clipped λ를 Q 스케일링에 그대로 사용하면 신호가 왜곡된다:

```
SPY 하락 → lam_raw 음수  →  clip 후 lam = 0.5  →  lam_scale = 0.2
→ Q = 0.2 × Q_OPTIMAL  →  시장이 하락해도 Q가 0에 못 닿음
```

따라서 **π용 λ와 Q용 λ를 분리**:
- `lam` (clipped): π 계산에만 사용
- `lam_raw` (unclipped): Q 스케일링에 사용

**동작**:

```
정상 국면:         lam_raw 양수, 크기에 따라 Q 자연 스케일
고변동성 진행 중:  SPY 하락 → lam_raw 음수 → max(0,...) = 0  (자연 도달)
고변동성 끝물:     시장 점진적 회복 → lam_raw 소폭 양수 → Q 자연 증가 시작
```

**Regime+λ(하드스탑)과의 비교**:

| 구간 | lam_raw | Regime+λ | raw_lam_Q | 어느 게 유리 |
|---|---|---|---|---|
| 폭락 진행 (SPY 하락) | 음수 | Q=0 | Q=0 (동일) | 동일 |
| 고변동성 끝물 + 점진적 회복 | 소폭 양수 | Q=0 (하드스탑 유지) | Q 자연 증가 | **raw_lam_Q 유리** |
| 급반등 (V자 반등) | 큰 양수 | Q=0 (하드스탑) | Q 상당히 양수 | **Regime+λ 유리** |

핵심 차이: 고변동성 끝물에서 시장이 점진적으로 회복 시작할 때, 하드스탑은 레짐이 바뀔 때까지 Q=0을 유지하고, raw_lam_Q는 λ 방향에 따라 자연스럽게 Q를 올리기 시작한다.

**장점**:
- Q가 자연스럽게 0에 도달 가능 → q_lambda의 "항상 Q > 0" 문제 해결
- 하드스탑처럼 이진 전환 없이 연속적 조정
- λ 부호 = SPY 초과수익 부호 → 직관적인 시장 신호

**한계**:
- 급반등 구간에서 lam_raw 양수 → Q > 0 → 고변동성 자산 반등 시 잘못된 방향 뷰 유지 (하드스탑이 잡는 케이스를 못 잡음)
- 상한 clip 없음 → 저변동성 + 강세장에서 lam_raw가 매우 커질 경우 Q 폭증 가능

**성과**: 미측정 (신규)

### 4-9. π_ratio

```python
def q_pi_ratio(P, pi, q_base, spread_ref):
    spread = abs(float(P @ pi))         # 현재 CAPM이 암시하는 저위험-고위험 스프레드
    spread_ref = expanding_median(...)  # 과거 스프레드 중앙값 (look-ahead bias 없음)
    return q_base * np.clip(spread / spread_ref, 0.1, 3.0)
```

**의미**: CAPM 균형(π)이 저위험-고위험 간 스프레드를 크게 부여하는 시기에는 Q도 함께 키운다.  
"CAPM이 이미 저위험 프리미엄을 인정할 때 내 뷰도 강하게."

**장점**:
- 역사적 실현 스프레드 사용 안 함 → 음수 문제 없음
- CAPM 균형과 Q가 동일 방향으로 움직임 → 이론적 일관성
- MDD 최소(-17.82%)

**한계**: P·π 절댓값은 λ, Σ에 동시 의존해 변동이 크다. 스프레드가 작을 때 Q도 작아져 뷰가 약해지는 구조가 항상 옳지는 않다.

**성과**: Sharpe 0.910, MDD -17.82% (MDD 최소)

## 5. 신규 Q 방식 아이디어

> **Q = 뷰의 점추정값** ("저위험 포트폴리오가 낼 기대수익률")  
> 이 섹션의 방식은 기대수익 크기를 직접 추정하는 신호를 사용한다.  
> 모델 신뢰도·신호 정확도 같은 불확실성 신호는 Ω에 반영해야 한다 → Section 6 참조.

### 5-1. Q_vol_spread — 변동성 예측 모델을 Q에 직접 연결

**핵심 아이디어**: 이미 GARCH로 예측한 `vol_predicted`가 있다. 저위험-고위험 그룹의 예측 변동성 격차가 클수록 다음 달 수익 스프레드도 클 것이라는 가설을 검증하고 Q를 추정한다.

```python
# 매 리밸런싱 시점 t에서:
vol_low_pred  = vol_pred[P > 0].mean()           # 저위험 그룹 예측 변동성
vol_high_pred = vol_pred[P < 0].mean()           # 고위험 그룹 예측 변동성
vol_spread_t  = vol_high_pred - vol_low_pred     # 예측 변동성 격차 (항상 양수)

# 학습 기간에서 회귀 추정 (walk-forward, look-ahead bias 없음):
# actual_ret_spread_t+1 = α + β × vol_spread_t + ε
# Q_t = α̂ + β̂ × vol_spread_t
```

**왜 유망한가**:
- 음수 문제 없음: vol_spread는 항상 양수 → Q 항상 양수 방향
- 파이프라인 일관성: GARCH vol 예측이 P 구성에도, Q 추정에도 동시 사용됨
- "변동성 예측을 잘할수록 Q 추정도 정교해진다"는 자기강화 구조

**한계**: α, β 추정에 충분한 학습 기간이 필요. vol_spread → ret_spread 관계 자체가 기간에 따라 달라질 수 있음. β가 불안정하거나 음수가 되면 Q 음수 위험 존재.

**대안 구현 — 회귀 없이 정규화 스케일러로 사용**:

회귀 추정 없이 vol_spread를 Q_lambda/π_ratio와 동일한 방식(고정 기준 대비 비율)으로 사용하면 α, β 불안정 문제를 피할 수 있다.

```python
# vol_spread: 매 시점 고위험 그룹 예측 vol - 저위험 그룹 예측 vol (항상 양수)
vol_spread_ref = vol_spread_series.expanding().median()  # look-ahead bias 없음
Q = Q_OPTIMAL * np.clip(vol_spread_t / vol_spread_ref, 0.1, 3.0)
# vol 격차가 역사적 중앙값보다 크면 Q 강화, 작으면 Q 약화
```

| | 회귀 버전 | 스케일러 버전 |
|--|-----------|--------------|
| Q 음수 위험 | 있음 (β < 0 가능) | 없음 (clip 0.1 하한) |
| 구현 복잡도 | 높음 (rolling OLS) | 낮음 |
| 수익률 단위 보정 | 있음 (α̂ + β̂ × X) | 없음 (Q_OPTIMAL에 의존) |
| Q_lambda와의 유사성 | 낮음 | 높음 — 동일 구조 |

**주의 — 위기 시 역방향 문제**: 위기 국면에서 vol_spread가 오히려 확대되면 Q가 증가하는 역방향 반응이 발생한다. Regime3의 하드스탑을 결합하면 이 문제가 원천 차단된다.

```python
Q = 0.0 if regime == '고변동성' else Q_OPTIMAL * np.clip(vol_spread_t / vol_spread_ref, 0.1, 3.0)
```

이 구조는 Regime+λ와 동일 (하드스탑 + 연속 스케일러), 신호 소스만 λ 대신 vol_spread로 교체한 형태다.

## 6. 신규 Ω 방식 아이디어

> **Ω = 뷰의 불확실성** ("Q를 얼마나 믿느냐")  
> 이 섹션의 방식은 모델 신뢰도·신호 강도를 불확실성 파라미터에 반영한다.

### Q vs Ω: 파라미터 역할 구분

BL 수식에서 Q와 Ω는 역할이 명확히 다르다:

| 파라미터 | 역할 | 올바른 신호 종류 |
|---------|------|----------------|
| **Q** | 뷰의 **점추정값** | 기대수익 크기를 추정하는 신호 (λ, vol_spread 등) |
| **Ω** | 뷰의 **불확실성** | 모델 정확도·신호 강도를 반영하는 신호 (GARCH 신뢰도, IC 등) |

의사 예시로 설명하면:
- Q = "이 환자의 기대 수명은 5년이다" ← **점추정값**
- Ω = "이 예측의 신뢰구간은 ±1년이다" ← **불확실성**

"모델이 정확하지 않으니까 기대 수명 추정을 3년으로 낮추자"가 아니라, "모델이 정확하지 않으니까 신뢰구간을 ±3년으로 넓히자"가 BL 프레임워크에 이론적으로 맞는 처리다.

**결론**: IC, GARCH_conf, Rank IC처럼 "예측력·신뢰도"를 나타내는 신호는 Q가 아니라 Ω에 반영해야 한다.

---

### 6-1. Ω_dynamic — GARCH 신뢰도를 Ω에 반영 ★ BL 이론 정합

```python
base_Omega = tau * float(P @ Sigma @ P)   # He-Litterman 기본 불확실성
conf = garch_conf_norm                     # [0.3, 1.0] — GARCH MAPE 기반 신뢰도
Omega_dynamic = base_Omega / conf         # 신뢰도 낮으면 Ω ↑ → BL이 Q 덜 반영
```

**의미**: GARCH 모델의 변동성 예측 정확도가 낮을수록 Ω를 키워 BL이 Q를 덜 반영하게 만든다.

```
Q는 고정, Ω만 조절 → 뷰의 점추정값은 유지, 신뢰도만 변경
신뢰도 높음 (conf=1.0) → Ω = base_Ω              (Q 방향으로 강하게 이동)
신뢰도 낮음 (conf=0.3) → Ω = base_Ω / 0.3 ≈ 3.3 × base_Ω (Q 방향으로 약하게 이동)
```

| 신뢰도 | Ω_dynamic | BL 조정 크기 |
|--------|-----------|-------------|
| 1.0 (최고) | base_Ω × 1.0 | 최대 — Q 방향으로 강하게 이동 |
| 0.5 | base_Ω × 2.0 | 절반 — Q 방향으로 약하게 이동 |
| 0.3 (최저) | base_Ω × 3.3 | 1/3 — Q 방향으로 매우 약하게 이동 |

**왜 이론적으로 더 타당한가**:
- Q를 줄이는 방식: "예측값 자체가 낮다"는 의미로 오해 가능
- Ω를 키우는 방식: "예측값은 그대로지만, 덜 믿는다"는 올바른 경제적 해석 유지

**한계**: Ω를 크게 키우면 BL 조정량이 급격히 감소 → Q=0과 유사한 효과. clip 범위 설계 필요.

---

### 6-2. Ω_IC — IC를 Ω에 반영

```python
# 12개월 롤링 IC (vol vs 다음 달 수익률의 Spearman 상관)
# 저위험 이상현상이 작동할 때: vol↓ → ret↑ → IC < 0
ic_rolling = ic_df['ic'].rolling(12).mean()

# IC를 신뢰도로 변환: 이상현상 강할수록(IC 더 음수) 신뢰도 높음
conf_ic = (-ic_rolling).clip(0.1, 1.0)  # IC < 0 → -IC > 0, 최소 0.1 (Ω 최대 10×)

base_Omega = tau * float(P @ Sigma @ P)
Omega_IC = base_Omega / conf_ic  # 이상현상 강함 → Ω 작음 → BL이 Q 강하게 반영
```

**의미**: 저위험 신호(vol)가 수익률을 잘 예측하는 기간일수록 Ω를 줄여 Q를 더 강하게 반영한다.

```
이상현상 강함 (IC = -0.5) → conf = 0.5 → Ω = 2×base_Ω  → Q 강하게 반영 ✅
이상현상 약함 (IC ≈ 0)   → conf ≈ 0.1 → Ω = 10×base_Ω → Q 거의 무시  ✅
이상현상 역전 (IC > 0)   → conf = 0.1 → Ω = 10×base_Ω → Q 거의 무시  ✅
```

**Q_IC_weighted 대비 Ω_IC가 이론적으로 올바른 이유**:
- Q_IC_weighted: IC로 Q를 줄임 → "기대수익이 낮다"는 의미로 오해 가능
- Ω_IC: IC로 Ω를 키움 → "뷰를 덜 믿는다"는 올바른 해석 유지

**Q_shrink와의 관계**: Q_shrink의 α=IC 연동 로직은 신호 강도에 따라 CAPM 쪽으로 수축한다. IC가 낮을 때 CAPM으로 후퇴하는 효과는 Ω_IC가 Ω를 크게 키우는 것과 동일한 결과를 낸다. Ω_IC가 이론적으로 더 명시적인 방식이다.

**한계**: IC 자체가 노이즈하고 월별 변동이 크다. 12개월 롤링으로도 불안정할 수 있어 추가 스무딩 필요.

---

### 6-3. Ω_RankIC — Rank IC를 Ω에 반영

```python
from scipy.stats import spearmanr

# vol 순위 vs 다음 달 수익률 순위의 Spearman 상관계수 (12개월 롤링)
ic_series = {}
for date in train_dates:
    vol_rank = vol_data.loc[date].rank()
    ret_next = ret_data.loc[date].rank()
    ic, _ = spearmanr(vol_rank, ret_next)
    ic_series[date] = ic

rank_ic_rolling = pd.Series(ic_series).rolling(12).mean()

# vol 순위가 수익률 순위를 잘 예측할수록 신뢰도 높음
conf_rank = (-rank_ic_rolling).clip(0.1, 1.0)

base_Omega = tau * float(P @ Sigma @ P)
Omega_RankIC = base_Omega / conf_rank
```

**의미**: vol 순위가 수익률 순위를 잘 예측할수록 Ω를 줄여 Q를 더 강하게 반영한다.

**Ω_IC vs Ω_RankIC 차이**:
- Ω_IC: vol 수준 vs 수익률의 Spearman 상관 (절댓값 기반)
- Ω_RankIC: vol **순위** vs 수익률 **순위**의 Spearman 상관 (이상치에 더 robust)

저위험 전략의 핵심은 "변동성 수준 예측"이 아니라 "변동성 순위 → 수익률 순위 예측력"이므로, **Ω_IC보다 Ω_RankIC가 전략 목적에 더 직접적인 지표다.**

**한계**: 동일하게 IC 노이즈 문제 존재.

**주의 — 이중계산(double-counting)**: Ω_IC와 Ω_RankIC는 같은 신호(IC)를 공유한다. 동시에 사용하면 이중계산이 발생하므로 하나만 선택해야 한다.

---

### Ω 방식 비교

| 방식 | 신호 소스 | 이론 정합성 | 구현 복잡도 | 권장 순위 |
|------|---------|-----------|-----------|---------|
| **Ω_dynamic** | GARCH MAPE | ✅ | 낮음 (conf 변수 기존 계산) | **1순위** |
| Ω_RankIC | vol순위~ret순위 Rank IC | ✅ | 중간 | 2순위 |
| Ω_IC | vol~ret IC | ✅ | 중간 | 2순위 (Ω_RankIC와 택1) |

**권장 조합**: `Q → Q_lambda` + `Ω → Ω_dynamic`
- 서로 다른 신호(λ vs GARCH_conf)를 사용하므로 이중계산 없음
- Q는 시장 위험회피(기대수익 크기), Ω는 모델 신뢰도(불확실성)로 역할 분리

## 7. Q 방식 평가 기준

Q의 역할을 한 문장으로 정의하면: **"현재 시점에서 저위험 이상현상에 대한 믿음의 강도."**  
Q는 수익률 예측값이 아니라 BL에 입력하는 확신의 크기다. 이 역할에서 도출되는 평가 기준은 다음 다섯 가지다.

---

### ① 방향 안전성 — Q ≥ 0이 구조적으로 보장되는가

**왜 중요한가**: P·π는 항상 음수다 (CAPM은 고변동 종목에 높은 기대수익을 부여하고, P는 고변동 그룹에 −를 곱하므로). 따라서 Q ≥ 0이면 diff = Q − P·π = Q + |P·π| > 0이 항상 보장된다.

```
Q ≥ 0  →  Q − P·π = Q + |P·π| > 0  →  저변동 우대 방향 유지 ✅
Q < 0  →  Q − P·π 가 음수가 될 수 있음  →  BL이 고위험으로 역전 ❌
```

**구조적 보장**의 의미: 수식이나 clip에 의해 Q < 0이 수학적으로 불가능한 경우.
- ✅ 구조적 보장: Q_lambda (clip 0.1 하한), π_ratio (abs 사용), Regime3/Regime+λ (0 또는 양수), Q_vol_spread 스케일러 (clip 0.1 하한)
- △ 부분 보장: Q_hist (60M 평균은 드물게 음수)
- ❌ 보장 없음: Q_momentum (12M 스프레드 → 음수 빈번), Q_ff3 (팩터 회귀 → 극단 음수)

Q_momentum이 MDD -42%로 폭발한 직접 원인이 이 기준 위반이다.

---

### ② 크기 제어 — 과집중 유발하지 않는 범위로 제한되는가

**왜 중요한가**: Q가 크면 BL이 저변동 종목의 기대수익을 크게 올리고, MVO가 소수 저변동 종목에 집중 투자한다. 개별 종목이 저변동이어도 소수에 집중하면 포트폴리오 자체는 고위험이 된다.

```
Q 커짐 → μ_BL(저변동) 급등 → MVO 과집중 → 포트폴리오 변동성 급등 → Sharpe 하락
```

**제어 방법**:
- **clip 상한**: Q_lambda, π_ratio, Q_vol_spread 스케일러 등의 clip(0.1, 3.0) → Q_OPTIMAL × 3.0 초과 불가
- **사전 grid search**: Q_FIXED = Q_OPTIMAL은 walk-forward에서 Sharpe 최대화 기준으로 탐색된 안전 범위 내 값
- **0으로 수렴**: Regime3/Regime+λ의 고변동성 Q=0 → 상한 문제가 아예 발생하지 않음

Q_hist가 실패한 이유 중 하나가 이 기준이다: 스프레드가 0 근처일 때 Q가 과소평가되어 과집중이 발생했다.

---

### ③ 시장 적응성 — 이상현상 약화 시 Q가 줄어드는가

**왜 중요한가**: 저위험 이상현상은 시장 환경에 따라 강도가 달라진다.

```
이상현상 강함: 고불확실성, 약세장, 일반 국면 → Q 크게 유지 ✅
이상현상 약함: 테크 강세장, 위기 초반 → Q 줄여야 방어 가능 ✅
이상현상 역전: 고위험 종목이 오히려 강세 → Q ≈ 0 또는 Q=0 ✅
```

**시장 적응 신호의 종류**:
- λ (위험회피계수): 시장 전체 리스크 선호도 반영 — 위기 시 λ 하락 → Q 자동 감소
- SPY 레짐 (Regime3): 변동성 분위수로 국면 분류 → 고변동성에 Q=0 하드스탑
- P·π 스프레드 (π_ratio): CAPM이 부여하는 저위험 프리미엄 크기 추적
- vol_spread + 하드스탑: vol 격차로 Q 조절, 위기 시 하드스탑

Q_FIXED는 이 기준에서 ❌다. 위기에도 강세장에도 동일한 뷰 강도를 유지한다.

---

### ④ 이론 정합성 — BL 프레임워크 안에서 경제적 논리가 있는가

**왜 중요한가**: 실증 성능이 같다면 "왜 이 방식인가"를 설명할 수 있어야 한다. 프레젠테이션·발표 시 defensible한 방법론이 필요하다.

BL 수식에서 각 파라미터의 역할이 명확하다:

| 파라미터 | 역할 | 설계 시 함의 |
|---------|------|------------|
| Q | 뷰의 **점추정값** | 저위험 포트폴리오가 낼 기대수익 |
| Ω | 뷰의 **불확실성** | Q를 얼마나 믿는가 |
| π | CAPM **균형** | 뷰 없을 때의 기준선 |

- ✅ 이론 정합: Q_lambda (He & Litterman 2002의 λ를 직접 사용), π_ratio (CAPM 균형과 연동)
- △ 부분 정합: Regime3/Regime+λ (레짐 경계값이 자의적), Q_vol_spread (vol→Q 관계 가설 기반)
- ❌ 정합 부족: Q_ff3 (FF3가 저변동성 이상현상을 설명하지 않음)

---

### ⑤ 구현 안정성 — 파라미터 민감도·추정 노이즈가 낮은가

**왜 중요한가**: 파라미터를 조금만 바꿔도 결과가 크게 달라지거나, 추정 신호가 매달 크게 흔들리면 walk-forward 백테스트 결과를 신뢰할 수 없다.

**평가 요소**:
- **파라미터 수**: Q_lambda는 λ_mean 1개, Regime+λ는 레짐 분위수+λ_mean 2개 → 많을수록 과적합 위험
- **신호 분산**: λ는 상대적으로 안정적. vol_spread는 위기 시 급변 가능
- **추정 방법 안정성**: rolling OLS (Q_vol_spread 회귀 버전)는 60개 관측치로 β를 추정하므로 표준오차가 큼
- **재현 가능성**: 동일 데이터에서 동일 결과가 나와야 함 (expanding 통계 사용 시 보장)

**Q_OPTIMAL 탐색 방식과 OOS 신뢰성**:

Q_OPTIMAL은 `Q_CANDIDATES` 8개 중 어떤 고정 Q 값을 기준으로 쓸지 결정하는 하이퍼파라미터다. 선택 방식에 따라 성과 수치의 신뢰도가 달라진다.

| 탐색 방식 | 설명 | OOS 편향 |
|---------|------|---------|
| 전 기간 in-sample | 전 기간 수익률 보고 Q 소급 선택 | **사후 선택 편향 있음** (소폭) |
| expanding window | 매월 과거 누적 Sharpe로 Q 결정 | **없음** (진짜 OOS) |

현재 구현(`99_baseline.ipynb`)은 expanding window 방식을 사용한다. 초기 24개월은 `Q_FIXED=0.003`을 fallback으로 사용하며, 이후 매월 `sens_raw` 누적 결과에서 Sharpe 최대 후보를 자동 선택한다.

**방식 간 비교 유효성**: Q_OPTIMAL 편향은 모든 방식에 동일하게 적용되므로, 어느 방식이든 expanding window를 쓰면 방식 간 상대 비교와 절대 성과 수치 모두 신뢰 가능하다.

### Q 강도와 저변동 우대 효과

저변동 우대 전략 자체는 유효하다. BL(Q_FIXED=0.003)은 CAPM 대비 Sharpe가 높다:

```
CAPM (BL 없음): Sharpe 0.985
BL Q_FIXED:     Sharpe 1.144  ← 저변동 우대의 효과
```

그러나 Q가 커질수록 성능이 역전된다:

```
Q 커짐
→ BL이 저변동 종목 기대수익을 더 크게 올림
→ MVO가 저변동 종목에 과집중
→ 분산투자 효과 소멸
→ 포트폴리오 변동성 급등
→ Sharpe 하락, MDD 폭발
```

개별 종목이 저변동이어도 소수 종목에 집중하면 포트폴리오 자체는 고위험이 된다. **저변동 우대는 해야 하되, 과집중을 유발하지 않을 강도로 해야 한다.** Q_FIXED=0.003이 좋은 이유는 값이 정확해서가 아니라 "충분히 작아서" 이 경계를 넘지 않기 때문이다.

## 9. 권장 우선순위

### λ 기반 3종 비교: Q_lambda vs Regime+λ vs raw_lam_Q

세 방식은 모두 λ를 신호로 사용하지만 설계 철학이 다르다.

| | Q_lambda | Regime+λ | raw_lam_Q |
|---|---|---|---|
| λ 종류 | clipped [0.5, 10.0] | clipped [0.5, 10.0] | raw (음수 가능) |
| Q 최솟값 | 0.2 × Q_OPTIMAL | 0 (고변동성) / 0.2× (그 외) | 0 |
| 고변동성 처리 | lam_scale 감소 (Q 소폭) | 하드스탑 Q=0 | lam_raw 음수 시 Q=0 |
| 급반등 케이스 | Q 소폭 양수 (문제) | Q=0 (방어) | Q 양수 (문제) |
| 고변동성 끝물 회복 | Q 최솟값 고정 (둔감) | Q=0 (과보수적) | Q 자연 증가 (유리) |

**백테스트 결과로만 판단 가능한 부분**: 급반등 vs 점진적 회복 중 어느 케이스가 더 자주, 더 크게 발생하는가.

1. **Q_vol_spread (스케일러 + 하드스탑)** — 가장 먼저 시도 권장
   - 이유: 이미 있는 `vol_predicted.csv`를 Q에 직접 연결. 파이프라인 목적(변동성 예측 → BL)과 가장 논리적으로 일관됨.
   - 구현 비용: 낮음 (vol_spread 계산 + expanding median + Regime 하드스탑)

#### Ω 방식

1. **Ω_dynamic** — 가장 먼저 시도 권장
   - 이유: Q를 변경하지 않고 불확실성만 조절 → BL 프레임워크에 이론적으로 가장 정확한 방식.
   - 구현 비용: 낮음 (garch_conf_norm 변수 기존 계산, Ω 산식만 변경)

2. **Ω_RankIC** — 두 번째 시도
   - 이유: 전략 핵심(vol 순위 → 수익률 순위 예측력)을 Ω에 직접 반영. Ω_dynamic과 병행하면 이중계산 없음 (신호 소스가 다름).
   - 구현 비용: 중간 (Rank IC 시계열 계산)

#### 권장 조합

```
Q = Q_lambda (또는 Regime+λ)
Ω = Ω_dynamic
```

- Q: 시장 위험회피계수 λ → 기대수익 강도 조절
- Ω: GARCH 신뢰도 → 뷰 불확실성 조절
- 두 신호가 서로 다른 정보를 사용 → 이중계산 없음

## 10. 추후 분석 항목

### 10-1. Prior를 1/n 동일가중으로 변경

현재 π는 시총 가중 포트폴리오를 역산한 CAPM 균형수익률이다. 이를 1/n 동일가중으로 바꾸면:

```python
w_prior = np.ones(n) / n          # 시총 가중 대신 동일가중
pi = lam * Sigma @ w_prior        # 동일가중 기반 균형수익률
```

**검토 이유**: 시총 가중 prior는 대형주에 편향된 균형을 가정한다. 1/n prior는 모든 종목에 동등한 사전 믿음을 부여하므로 소형주·저변동 종목의 μ_BL 조정이 더 고르게 분산될 수 있다.

**확인할 것**: π 절댓값이 달라지므로 P·π 부호 및 크기가 바뀜 → Q_FIXED 재탐색 필요.

---

### 10-2. P 행렬 가중치를 변동성 역수로 변경

현재 P는 저위험/고위험 그룹 내 가중치를 시총으로 부여한다. 이를 변동성 역수로 바꾸면:

```python
w_i ∝ 1 / sigma_i    # 변동성 낮을수록 P에서 더 큰 가중치
```

**검토 이유**: 저위험 이상현상의 논리(낮은 변동성 = 더 순수한 저위험 신호)와 직접 연결된다. Risk Parity 포트폴리오와 동일한 철학.

**주의**: π가 시총 기반으로 계산된 값이므로 P만 변동성 역수로 바꾸면 P와 π의 이론적 기반이 어긋난다 → π도 함께 1/n 또는 변동성 역수 기반으로 바꾸는 것을 병행 검토.
