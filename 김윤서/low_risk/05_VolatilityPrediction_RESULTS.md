## Part 0. QLIKE 손실함수 — 개념과 해석 방법

### QLIKE란?

**QLIKE (Quasi-Likelihood)** 는 변동성 예측 모델의 정확도를 측정하는 손실함수다.

$$\text{QLIKE}(\hat{h},\, \sigma^2) = \frac{\hat{h}}{\sigma^2} - \log\!\left(\frac{\hat{h}}{\sigma^2}\right) - 1$$

| 기호 | 의미 |
|------|------|
| $\hat{h} = \hat{\sigma}^2$ | GARCH가 예측한 분산 (예측 변동성²) |
| $\sigma^2$ | 실현 분산 (실현 변동성², `vol_21d`²) |
| 최솟값 0 | 완벽한 예측일 때 QLIKE = 0 |

---

### 왜 RMSE 대신 QLIKE를 쓰는가?

변동성은 직접 관찰할 수 없고, `vol_21d`(21일 롤링 표준편차) 같은 **노이즈 있는 대리변수(proxy)** 로만 측정된다.

Patton (2011) 논문이 수학적으로 증명한 핵심 결과:

> RMSE는 대리변수의 노이즈가 있으면 잘못된 모델을 선택할 수 있다.  
> QLIKE는 대리변수 노이즈 하에서도 올바른 모델 비교를 보장하는 **강건한(consistent) 손실함수**다.

따라서 변동성 예측 논문의 표준 성능 지표는 RMSE가 아닌 QLIKE다.

---

### 어떻게 해석하는가?

#### ① 작을수록 좋다 (최솟값 = 0)

| QLIKE 값 | 의미 |
|----------|------|
| 0 | 완벽한 예측 (예측 = 실현) |
| 0.2 ~ 0.5 | 실전에서 일반적인 정상 범위 |
| 1 이상 | 예측 품질 저하 신호 |
| 10 이상 | 이상치 또는 데이터 결함 (SW 수정 전 발생한 케이스) |

#### ② 비대칭 페널티 — 과대예측이 더 나쁘다

ratio = $\hat{h} / \sigma^2$ (예측분산 / 실현분산)으로 생각하면:

| 상황 | ratio | QLIKE | 해석 |
|------|-------|-------|------|
| 과소예측 (예측 < 실현) | < 1 | 작음 | 패널티 상대적으로 작음 |
| 완벽 예측 | = 1 | 0 | 최적 |
| 과대예측 (예측 > 실현) | > 1 | 급등 | 페널티 강함 |

**직관**: "변동성을 낮게 예측했다가 실제로 크게 흔들리는 것"보다 "변동성을 높게 예측했다가 실제로 잠잠한 것"에 더 강한 페널티를 준다. 리스크 관리 관점에서 과소예측(과소헤지)이 실제로 더 위험하지 않냐고 생각할 수 있지만, QLIKE의 페널티 구조는 수학적 도출 결과이지 직관적 선호가 아니다.

#### ③ 2020년 COVID에서 QLIKE가 낮게 나온 이유

2020년 RMSE = 0.2665(최고)였지만 QLIKE = 0.3983으로 평균 수준이었다. 이는 QLIKE의 비대칭 페널티 때문이다:

```
2020-03: GARCH가 낮은 vol 예측 (저변동성 훈련 데이터 기준)
실제: COVID → 실현 vol 폭등 → ratio = ĥ/σ² << 1 (과소예측)
QLIKE: 과소예측 구간 → 페널티 작음 → 낮은 QLIKE 값
RMSE:  |예측 - 실현| 방향 무관 → 오차 그대로 반영 → 높은 값
```

#### ④ GARCH vs Baseline QLIKE 비교 해석

| | GARCH | Baseline(AR0) | 우위 |
|-|-------|--------------|------|
| QLIKE 평균 | **0.4399** | 0.4899 | **GARCH** |
| QLIKE 중앙값 | 0.3804 | **0.3764** | Baseline (차이 0.004) |
| QLIKE 표준편차 | **0.2705** | 0.3890 | **GARCH** |
| Rank IC 평균 | **0.5876** | 0.5682 | **GARCH** |

QLIKE 평균과 표준편차 기준 GARCH가 우세, 중앙값은 Baseline이 0.004 차이로 소폭 우세.  
중앙값 역전 이유: 레짐 전환 달에 Baseline의 오차가 크게 튀어 평균을 끌어올리기 때문이다.  
포트폴리오에서 필요한 것은 "어떤 종목이 더 위험한가"(순위)이므로 **Rank IC가 핵심 지표**이며, 종합적으로 GARCH가 Baseline보다 우수하다.

```

  → 2011-10: 월 QLIKE 66,678,989 (단일 종목)
  → 2016-08: 월 QLIKE 6,517,118 (단일 종목)
```

### Rank IC 해석 주의사항 — 변동성 예측 vs 수익률 예측

#### IC=0.05 기준은 수익률 예측 맥락의 수치다

팩터 투자 문헌(Grinold & Kahn 등)에서 IC=0.05를 "유의미"로 보는 이유는, 주가 수익률이 거의 랜덤워크에 가까워 예측력 없는 모델은 IC≈0이기 때문이다.

```
수익률 예측:  기저 IC ≈ 0     → 0.05 이상이면 "의미 있음"
변동성 예측:  기저 IC ≈ 0.57  → 0.57 이상이어야 "의미 있음"
```

변동성은 자기상관이 높고(변동성 군집 효과), 직전 달 vol을 그대로 이월하는 Baseline만으로도 Rank IC=0.5682가 나온다. 따라서 변동성 예측의 IC 임계값은 수익률 예측보다 훨씬 까다롭다.

#### GARCH IC의 실질적 의미 분해

| 구성 요소 | IC | 출처 |
|---------|-----|------|
| 전체 GARCH IC | 0.5876 | 변동성 지속성 + GARCH 모형 기여 |
| Baseline IC (직전 vol 이월) | 0.5682 | 변동성 자기상관(지속성)만으로 설명 |
| **GARCH 순수 기여분** | **+0.0194** | GARCH 모형이 Baseline 대비 추가한 예측력 |

전체 IC 0.5876의 약 **97%는 "변동성이 원래 지속되는 성질"** 때문이고, GARCH 모형 자체의 기여는 +0.019(약 3%)다.

#### 그럼 GARCH를 쓰는 의미가 없는 것인가?

그렇지 않다. +0.019의 절댓값이 작아도 **그 기여의 안정성(ICIR)** 이 매우 높다:

| | GARCH | Baseline |
|--|-------|----------|
| Rank IC 평균 | 0.5876 | 0.5682 |
| ICIR | **7.26** | 5.09 |
| IC 양수 비율 | **100%** | **100%** |
| P Precision | **0.584** | 0.563 |

GARCH의 +0.019 기여는 180개월 내내 흔들림 없이 일관되게 발생한다(ICIR 7.26 vs Baseline 5.09). 변동성 분리도와 Precision에서도 GARCH가 체계적으로 우세하다. 따라서 "IC 0.05 기준"이 아닌 **"Baseline 대비 +0.019의 일관된 개선"** 이 GARCH 선택의 근거다.
